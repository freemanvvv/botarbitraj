# NormBase — локальный RAG по нормативам Узбекистана (Фаза 1)

Пайплайн скачивает/принимает строительные нормы (КМК / ШНК / СНиП / ГОСТ),
извлекает текст (с OCR для сканов), режет на чанки с метаданными (включая
**статус действия нормы**), считает эмбеддинги через LM Studio и кладёт в
локальную ChromaDB. Поиск возвращает фрагменты **с цитатой** (документ, пункт,
страница) и опционально — ответ LLM строго по найденному.

Всё работает локально на MacBook M3, без облака.

---

## 1. Установка

### Системные зависимости (Tesseract + языки)
```bash
brew install tesseract tesseract-lang
# проверь, что есть rus / uzb / uzb_cyrl:
tesseract --list-langs | grep -E "rus|uzb"
```

### Python-окружение
```bash
cd normbase
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
```

### LM Studio
1. Установи LM Studio, включи **Apple Metal**.
2. Скачай и загрузи **embedding-модель** — рекомендуется **bge-m3**
   (мультиязычная, хорошо берёт русский и узбекский).
3. Включи локальный сервер (вкладка Developer → Start Server),
   адрес должен быть `http://localhost:1234`.
4. Посмотри точный **id модели** в LM Studio и впиши его в `config.py`
   → `EMBEDDING_MODEL`. Для ответов (`--ask`) загрузи ещё chat-модель
   (Qwen3-8B) и укажи её в `CHAT_MODEL`.

> На 20 ГБ не держи одновременно 14B-модель и индексацию. Для Фазы 1
> достаточно 8B + embedding-модели — памяти хватит с запасом.

## 2. Заполнение источников (`sources.csv`)

Каждая строка — один документ. Колонки:

| Колонка | Назначение |
|---|---|
| `id` | уникальный идентификатор (латиницей, без пробелов) |
| `doc_type` | КМК / ШНК / СНиП / ГОСТ |
| `number`, `year`, `title` | реквизиты для цитаты |
| `language` | `ru` или `uz` (для двуязычной базы заводи документ дважды) |
| `status` | `active` / `superseded` / `unknown` |
| `superseded_by` | чем заменён (если отменён) |
| `source_url` | прямая ссылка на PDF/HTML (скачается автоматически) |
| `local_path` | путь к уже скачанному файлу (приоритетнее URL) |

В комплекте идёт стартовый `sources.csv` с примерами реальных КМК/ШНК.
**Статусы в примерах проверь сам** — см. раздел «Где брать и как проверять» ниже.

---

## 3. Запуск

```bash
# проиндексировать всё из sources.csv
python ingest.py

# для пробы — только 2 документа
python ingest.py --limit 2

# переиндексировать с нуля
python ingest.py --reset
```

Поиск:
```bash
python query.py "минимальная высота ступени лестницы"
python query.py "приёмка объекта в эксплуатацию" --n 8
python query.py "высота ограждения лестницы" --ask     # + ответ LLM по найденному
python query.py "..." --status all                     # включая отменённые нормы
```

По умолчанию из выдачи **исключаются отменённые нормы** (`status=superseded`).

## 4. Где брать документы и как проверять статус

**Официальные первоисточники Узбекистана**
- `mc.uz` — портал Министерства строительства (PDF самих КМК/ШНК).
- `shaharsozlik.uz` — градостроительные нормы по группам.
- `lex.uz` — Нацбаза законодательства; здесь акты чаще доступны **и на
  русском, и на узбекском** — основной канал для двуязычности.

**ГОСТ / стандарты**
- `standards.uz` (Узстандарт / Агентство техрегулирования).

**Российские СНиП / СП / ГОСТ** (где применяются в Уз)
- `docs.cntd.ru`, `meganorm.ru`, ФГИС Росстандарта.

**Проверка «действует / отменён»**
- `norma.uz` (система «Строительство») отслеживает актуальные ШНК/КМК с датами.
- Перед продакшном статус каждой нормы сверяй с lex.uz / norma.uz и проставляй
  в `sources.csv`. Это важнее качества модели: RAG не должен цитировать
  отменённые пункты.

## 5. Ограничения (честно)

- **Качество = качество источников.** Кривой OCR → кривой поиск. Сложные
  таблицы и чертежи внутри PDF распознаются плохо.
- **OCR — не понимание.** Текст со сканов вытащим, но геометрию чертежей нет
  (vision-модель в 20 ГБ не помещается — это вне Фазы 1).
- **Модель может ошибаться** даже по найденному — поэтому всегда показываем
  цитату, чтобы человек мог проверить первоисточник.
- **Лицензии.** Официальные издания формально нельзя тиражировать без
  разрешения Госкомархитектстроя. Для личного локального RAG это серая зона;
  для коммерческого продукта по этим текстам нужна проработка прав.

## 6. Структура проекта

```
normbase/
├── config.py        # все настройки (пути, LM Studio, чанкинг, OCR)
├── sources.csv      # манифест документов (заполняешь ты)
├── extract.py       # PDF/HTML → текст (+ OCR сканов)
├── chunk.py         # нарезка по пунктам нормативов с метаданными
├── embed.py         # эмбеддинги через LM Studio
├── store.py         # ChromaDB (постоянное хранилище)
├── ingest.py        # пайплайн индексации (CLI)
├── query.py         # поиск + цитаты + опц. ответ LLM (CLI)
└── data/            # создаётся автоматически: raw/ text/ chroma/
```

## 7. Место в общем плане

Это исполнение блока «RAG по нормативам» из **Фазы 1**. Дальше к нему
пристыковываются генерация документов (ТЗ, пояснительные записки в PDF/DOCX)
и маршрутизация моделей RU/UZ. Поиск уже отдаёт цитаты с пунктами — на этом
строится честный ответ ассистента без «галлюцинаций» по нормам.
